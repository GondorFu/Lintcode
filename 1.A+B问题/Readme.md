# GondorFu
---
## 描述
给出两个整数a和b, 求他们的和, 但不能使用 + 等数学运算符。

## 说明
```
a和b都是 32位 整数么？

是的
我可以使用位运算符么？

当然可以
```

## 样例
```
如果 a=1 并且 b=2，返回3
```

## 挑战 
显然你可以直接 return a + b，但是你是否可以挑战一下不这样做？

## 标签 
Cracking The Coding Interview 比特位操作

## 解题思路
- 思路：
    
    凡是类似的题都应该想到位运算，尤其是异或运算

- 具体操作：
    
    a异或b能够解决0和1之间的加，a与b并向左移动能够解决1和1运算的进位。之后就是简单处理一下边界条件，即a和b有一个为零的情况。

## 特别说明

    发现直接将算法移植到Python中无法运行。

    最后发现原因是Python没有声明变量的类型，所以在移位运算时，并未将最高位移出。因此b一直不能为零。最后通过ctypes类中的函数解决了这个问题。
