# GondorFu
---
## 描述
给出三个字符串:s1、s2、s3，判断s3是否由s1和s2交叉构成。

## 样例
```
比如 s1 = "aabcc" s2 = "dbbca"

    - 当 s3 = "aadbbcbcac"，返回  true.

    - 当 s3 = "aadbbbaccc"， 返回 false.
```

## 挑战
要求时间复杂度为O(n^2)或者更好

## 标签 
动态规划 最长公共子串

## 解题思路
- 思路：
    
    由于如果出现s1与s2同时匹配s3，无法判断用哪一个进行匹配，所以只能使用动态规划。
    
    状态转移方程:
    
    A[m, n] = (A[m-1, n] && s1[m-1] == s3[m+n-1]) ||  (A[m, n-1] && s2[n-1] == s3[m+n-1])
    
    A[m, n]表示s1取前m个元素，s2取前n个元素，与s3的前m+n个元素是否匹配。
    
- 具体操作：

    由于具体判断只涉及上一个字符的情况，所以可以用一维数据优化存储空间。
    
    首先初始化一个长度为n1+1的逻辑数组，除第一个元素为真其他为假。即A[0,0]=True.接着对s2,s1进行二重遍历，注意边界条件，最后的l[-1]就是结果。
