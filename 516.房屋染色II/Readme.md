# GondorFu
---
## 描述
这里有n个房子在一列直线上，现在我们需要给房屋染色，分别有红色蓝色和绿色。每个房屋染不同的颜色费用也不同，你需要设计一种染色方案使得相邻的房屋颜色不同，并且费用最小。

费用通过一个nx3 的矩阵给出，比如cost[0][0]表示房屋0染红色的费用，cost[1][2]表示房屋1染绿色的费用。

## 注意事项
所有费用都是正整数

## 样例
```
costs = [[14,2,11],[11,14,5],[14,3,10]] return 10

房屋 0 蓝色, 房屋 1 绿色, 房屋 2 蓝色， 2 + 5 + 3 = 10
```

## 挑战
用O(nk)的时间复杂度解决

## 标签
动态规划 脸书

## 解题思路
- 思路：
    
    具体动态规划分析参考515题。不过这里需要一个额外的优化，不然超时。
    
    进过分析可以发现对于每种颜色并不需要每次都去枚举除去这种颜色之后的最小代价，特别是本题其实并不限制颜色为三种，可以很多。
    
- 具体操作：

    对于n-1个房屋求得的最小代价数据，直接找出它的最小代价，如果最小代价数量不唯一，那么第n个房屋都可以选择n-1房屋的最小代价作为前一个房屋的最小代价。如果唯一，那么对应颜色的n-1房屋的最小代价将为除去这个最小代价之后的最小代价，其他颜色不变。具体操作见程序。
