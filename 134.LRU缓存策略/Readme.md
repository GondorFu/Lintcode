## 描述
为最近最少使用（LRU）缓存策略设计一个数据结构，它应该支持以下操作：获取数据（get）和写入数据（set）。

获取数据get(key)：如果缓存中存在key，则获取其数据值（通常是正数），否则返回-1。

写入数据set(key, value)：如果key还没有在缓存中，则写入其数据值。当缓存达到上限，它应该在写入新数据之前删除最近最少使用的数据用来腾出空闲位置。

## 解题思路
- 思路：
    
    Cache管理的一个经典应用实例。Cache管理的两个经典方法是LRU（Least Recently Used）和LFU（Least Frequently Used，24.LFU缓存）。LRU就是将使用时间最久之前的数据先删除，而LFU就是将使用次数最少的数据先删除。
    
    LRU的基本思路是将所有数据使用一个链表进行存储，同时为了更高效的删除添加元素，通常使用双向链表，并添加头尾两个指针。头指针指向使用时间最久的数据，尾指针指向最新使用过的数据。同时，为了高效地查找数据，可以使用哈希记录每个数据的位置。
    
- 具体操作：

    起始时构建如上数据结构。当获取数据时，通过哈希查找，如果不存在直接返回-1，反之返回对应数据，并将该数据移动到链表尾指针处。当写入数据时，同样通过哈希查找，如果不存在在尾指针处插入该数据，如果存在则修改数据值，并将数据移动到尾指针处。在这个过程中使用使用哈希存储数据的位置。
