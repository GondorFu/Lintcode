# GondorFu
---
## 描述：
给定一个整数数组，找出两个不重叠的子数组A和B，使两个子数组和的差的绝对值|SUM(A) - SUM(B)|最大。

返回这个最大的差值。

- 注意事项

    子数组最少包含一个数

## 样例
```
给出数组[1, 2, -3, 1]，返回 6
```

## 挑战 
时间复杂度为O(n)，空间复杂度为O(n)

## 标签 
贪心 枚举法 数组 子数组 前后遍历

## 解题思路
- 思想：
    这是一道逻辑思考题，在找到解法之前需要仔细分析结果所具有的规律。
    
    首先需要想到如下特点：最大子数组和最小子数组一定是首尾相接的。
    为了简单解释可以想象两个子数组之间只有一个数c，即[max, c, min]。此时，如果c大于0，那么max就应该包含c。如果c小于0，那么min就应该包含c。如果c等于0，那么无论max或者min包含c都不会影响最后的结果。对于中间包含更多数的情况可以类似分析。
    
    依据前面所说的性质，我们就可以依此遍历数组中的每一个位置，比如样例中的[1,2,-3,1]。由于子数组必须包含一个数，所以位置只能选1、2两个。当选i时，意味着[0:i]和[i+1:-1]两个数组中选取两个子数组让它们之差最大。又由于两个数组必须相邻，所以左边的数组必须包含i，右边的数组必须包含i+1。
    
    这样，最大最小数组就能很方便的求解了。以从左开始的最大数组为例，lmax[i]表示数组[0:i]上最大子数组的值。i=0时，lmax[i]=1；i=1时，由于前面的分析i必须包含在最大子数组中，所以如果lmax[i-1]大于0，那么lmax[i]=lmax[i-1]+nums[i];否则，lmax[i]=nums[i]。
    
    依次计算lmax, lmin, rmax, rmin，然后遍历划分就能找到最大差值。
    
- 具体操作：

    思路中已经介绍的比较清楚。大概的步骤就是从左遍历数组元素，按照介绍的方法求解lmin和lmax。然后从右遍历数组元素，求解rmin和rmax。最后遍历划分，计算lmax[i]-rmin[n-i]和rmax[n-1]-lmin[i]得到较大值。最后得到整个数组的最大差值。
