# GondorFu
---
## 描述：
给出一个字符串 A, 表示一个 n 位正整数, 删除其中 k 位数字，使得剩余的数字仍然按照原来的顺序排列产生一个新的正整数。

找到删除 k 个数字之后的最小正整数。

N <= 240, k <= N

## 样例
```
给出一个字符串代表的正整数 A 和一个整数 k, 其中 A = 178542, k = 4

返回一个字符串 "12"
```

## 标签 
贪心

## 解题思路
- 思想：这是一道逻辑题，需要好好分析问题具有的特性，之后才能找的好的解题方法。

以'170896'为例，如果k=1,结果为'10896';如果k=2,结果为'896';如果k=3,结果为'86';可以发现要删除的顺序应该为从左到右最先出现数值下降的元素，第一个是7。删除7之后，由于0在1之后数值也出现了下降，所以下一个是1。以此类推。

可以看出每次都选取当前状态下的最优选择能够导致最后最优的结果，因此这是一道贪心算法的题目。
- 具体操作：为此，具体实现可以从左到右遍历字符串，如果出现数值下降则删除前面数值较大的元素。同时判断是否已经删除了足够的元素。如果遍历到最后，还需删除元素。能够发现此时字符串已经按从小到大排列。此时，只需不断删除末尾的元素就能得到最后的结果。

不过，需要考虑结果字符串以'0'开头的情况和一些边界条件，因此在最开始给字符串开头加上'0'。
